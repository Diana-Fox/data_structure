# 图

```
图包含顶点V，如果两个顶点存在对应关系则连边E。
彼此之间存在这种关系，并且存在连边的任何两个点，称为彼此连接的，称作邻接关系。描述顶点与顶点的关系。
参与定义邻接关系的每一个顶点与邻接关系之间的关系也称作关联关系。描述顶点与某条边之间的关系。
```

# 无向图/有向图

```
若邻接顶点u和v的次序无所谓，则（u,v)为无向边undirected edge
所有的边均无方向的图，即无向图undigraph
反之，有向图digraph中均为有向边directed edge
u、v分别称作边(u,v)的尾（tail）、头（head）
```

# 路径/环路

```
路径：由一系列的顶点按照依次连接的关系，构成的序列。
简单路径：不含重复节点的路径
环、环路：路径的起点和终点重合
有向无环图（DAG）不包含任何环路
Eulerian tour（欧拉环路）:经过所有的边一次，并且恰好一次的环路。
Hamiltonian tour（哈密尔顿环路）：经过每个顶点一次，并且恰好一次。

```

# 图的表示方法

## 邻接矩阵

```
用以描述顶点之间相互邻接关系的一种形式。
如果图中包含n个顶点，则矩阵为n*n。其中第i行第j列表示顶点与顶点之间是否存在邻边。
无向图对应的连接矩阵应该是对称的。对角线上应该是都连接，因为是自环，不过不做讨论。01的bit矩阵表示即可。
如果是带权图，每条边都拥有权重，则只需对本形式略加扩充，将权重存入对应单元即可。整数或浮点数即可。
```

## 关联矩阵

```
如果当前的图有n个节点，那么当前的图就有n行，如果图中总共包含e条边，则矩阵e列。
相应的，矩阵中的任何一个单元，表示的是对应的顶点与对应边是否存在关联关系。存在记为1，不存在记为0。
```

# 邻接矩阵实例

```
无向图（undigraph）：对称，存储效率冗余
有向图(digraph)：如果是单向边，则不对称。
带权图（network）：各边对应的权重记录于各单元。
```

# 邻接矩阵表示法

```
优点：
    直观,易于理解和实现
    适用范围广泛:digraph/network/cyclic/...
    尤其适用于稠密图(dense graph)
    判断两点之间是否存在联边:O(1)
    获取顶点的(出/入)度数:O(1)
    添加、删除边后更新度数：O(1)
    扩展性（scalability)：
        得益于Vertor良好的空间控制策略，空间溢出等情况可“透明地”予以处理。
    
缺点：
    n^2空间，与边数无关。
    平面图（planar graph)：可嵌入于平面的图（边不可相交）
        欧拉公式可以表示：v - e + f - c = 1
        即对于任何一个平面图，0维元素（即顶点）数目，以及1维元素e(边数），2维元素f（面数）以及连通域的总数（c)必然满足以上恒等式。
          平面图：e≤3xn-6=O(n)<<n^2,此时空间利用率≈1/n，趋近于0
```

# 广度优先搜索

```
化繁为简，将图转为树。
算法：
    始自顶点s的广度优先搜索Breadth-First_Search
    访问顶点s
    依次访问s所有尚未访问的邻接顶点
    依次访问它们尚未访问的邻接顶点
    。。。。。。
    如此反复
    直到没有尚未访问的邻接顶点
```
# 深度优先搜索
```
DFS(s) //始自顶点s的深度优先搜索（Depth-First Search)
访问顶点s
若s尚有未被访问的邻居，则任取其一u，递归执行DFS(u)
否则，返回
```
